Learning the Basics#

Once you have learned some basic components of the C++ language, understanding the language becomes much easier, because you will be able to write simple programmes to try out each new concept.

This page leads you through some basic exercises, much of the explanation is left until later chapters - this page is not intended to present the full syntax.

In the Computational Physics module, we are mainly concerned with writing programmes that will take user inputs, perform calculations, and then return the ‘answers’ to the user, which requires us to learn only a subset of the C++ language and programming concepts.

Expressions and Statements#

A computer programme is a sequence of statements, which are executed sequentially when the programme runs.

Types of Statement

C++ includes the following types of statements:

  • Labelled statements - adds a label to another type of statement, to allow control of the flow.

  • Expression statements

  • Compound statements

  • Selection statements

  • Iteration statements

  • [Jump statements]

The simplest type of statement is an expression statement - an expression followed by a terminator.

Expressions are sequences of operators and their operands, which specify a computation.

When the expression statement is executed, the expression is evaluated by the computer, producing a result, which will have a value and a type.

Statements are always terminated by either a semicolon ; or by a code-block {…some code here…}, which may be empty or contain statements.

Tip

Only directives do not require termination. For example: `#include

Tip

The Visual Studio IDE indicates a missing ; by highlighting the start of the next line of code.

Forgetting to terminate a statement, will result in a compile-time error.

A semicolon is a statement terminator, and by itself a statement, an empty statement (null statement).

An expression statement without an expression is called a null statement, which may be used to provide an empty body to a for or while loop, or to carry a label in the end of a compound statement.

Basic Arithmetic Operations#

All programming languages carry out basic arithmetic operations. Normally we do not use a computer as a simple calculator, but most programmes carry out some arithmetic operations as part of their execution.

std::cout can be used to demonstrate the basic arithmetic operations.

Variables#

A computer programme that can only perform arithmetic on literal values written into the original programme code would not be very useful.

Instead, programmes must be able ask for the numeric values that are to be used in the calculations.

This requires the computer to store the number, and to be able to identify and manipulate it later.

Computers use variables for this purpose.

At its simplest: like in maths, the variable is a name that represents a value that will be stored by the computer.

To get started we will look at creating two types of variables.

To create a variable named a that holds the integer value of ‘5’, we can used the statement int a = 5;.

After this statement in the programme, the variable a may be used anywhere (see scope) that we want to use its value of 5.

Strings#

Another useful type of variable is the string type. A string is a series of individual characters.

To use string-type variables we need an additional component of the standard library, <string> to be included.

The string literal must be enclosed in "".

More Data Types#

In addition to strings, and integers, we need variable from the floating-point number types and char type.

Code Explanation

Single quotes denote a char type.

Float and double are both used for floating-point numbers - “double” denoting double precision (twice as many bits required for their representation in binary).

Scientific notation is formatted using a lower or upper case e between the significant and the index WITHOUT ANY SPACES!!

int is the keyword for the integer type.

int may be preceded by one or two modifiers - signed, unsigned, short, long, long long. The modifiers may be used on their own, and intis implied; butsignedandunsignedmay also precedechar, and long may precede double.

Escape sequences, or ‘escape characters’, are used to represent special characters within a string or character literal.

Certain characters in the program must be represented with escape sequences because they are reserved characters that have specific meaning to the compiler.

Each escape character is identified to the program by prefixing \ to the character.

These are mostly used in formatting output. A full list of escape characters in C++ can be found here.
